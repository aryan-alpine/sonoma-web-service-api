Alpine REST API


Purpose

The purpose of this documentation is to provide client developers (iPhone, android, other)
with everything they need to be able to use Alpine RESTful services in their apps.

Pre-requisite knowledge

This service specification assumes that the reader has a solid grounding in the pre-requisite technologies.

Common behaviors

All alpine services exhibit certain common behaviors.
Users should familiarize themselves with these common behaviors.

Recent changes

Recent changes made to the API.

Cross-Application

RESTful Services

These services are not application specific and can be used by all applications.
https://test.aosvapps.com/[serviceName]
Example:
https://test.aosvapps.com/account

Method Resource Description Auth Req'd?
POST account Create a user account. None
GET account Read the details of a specific user account. User
GET account/password Reset the password for a specific user account. None
PUT account Update the details of a specific user account. User
DELETE account Remove a specific user account. User
POST deviceLogEntries2 Post one or more device log entries to the server.
Version 2 of the logging format.		 Device (future)
POST image Post a base64 encoded image to the server. Device (future)
GET image Retrieve an image that has been stored uploaded to the server. None
GET messages GET a list of messages for a particular application for a particular user of a particular type. User
POST message Send a message to one or more recipients via one or more transport mechanisms {sms, email, android, iphone}. User

Method Resource Description Auth Req'd?
POST vehicles Associate one or more vehicles -
each with a specific make, model and year -
to a particular Google Cloud Messaging registration id.
This will enable us to send them notifications of recalls.		 NO
GET vehicles Get the vehicles associated to a particular GCM registration id. NO
PUT vehicles Update the list of vehicles associated with a particular GCM registration id. NO
DELETE vehicles Remove a GCM registration id and the list of vehicles associated with it. NO
POST recalls ADMIN ONLY
Add new recall a list of one or more recalls.
For each individual recall - for a specific vehicle (make, model, year) -
send a GCM notification to everyone affected.		 YES
PUT recall ADMIN ONLY
Update an individual recall record.		 YES
GET recall Get the details of a specific recall notification.
This is the service that client should call for each GCM notification it receives.
The GCM notification will contain the recordId that should be used when calling this service.		 NO
GET recalls Get the list of recalls for a vehicle with a specific make, model and year. NO
DELETE recall ADMIN ONLY
Delete a specific recall. 		YES
