NOTIFICATION: New Recall

A new recall has been issued for your vehicle.


Google Cloud Messaging (GCM) Message Format

{
    "recordId": 123456789,
    "make": "FORD",
    "model": "MUSTANG",
    "year": "2007"
}

The user is receiving this notification because he/she selected to register one or more vehicles by calling the POST vehicles service.

In order to stop receiving these messages, the user must change the list of vehicles associated with his/her account by calling the DELETE vehicles service.

The notification gets sent to the user when an administrator invokes the POST recalls service and a MATCH is discovered between the vehicle the user registered and a vehicle listed in one of the recalls posted.

When the user receives this notification, the app developer will typically want to retrieve the details of the new recall by invoking the GET recall service. The recordId received in the GCM notification is the one that should be used to call the GET recall service.

The reason that the make, model & year is delivered in addition to the recordId is that the user *might* select to register more than a single vehicle. It therefore might be useful to the app developer to know which of these multiple vehicles the new recall has been issued for.
