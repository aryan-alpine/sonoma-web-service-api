Alpine REST API



Pre-requisites

In order to understand this documentation, you must have a solid understanding of:

(1) HTTP: Hypertext Transport Protocol
(2) REST: Representational State Transfer
(3) JSON: JavaScript Object Notation

(1) HTTP: Hypertext Transport Protocol

You must know, for example:

HTTP requests and responses are both documents that are structured into 2 parts:
(1) HTTP Headers
(2) HTTP Body

(1) HTTP Headers:

A collection of name/value pairs specified by the HTTP specification.

Request Headers

You should be familiar with the most common types of HTTP Headers used in HTTP requests ...
The most important of the request headers is the request method
Request method (mandatory)
Example request methods include: POST, GET, PUT & DELETE
(Note: In REST these methods correspond to CREATE, READ, UPDATE & DELETE: CRUD)

Response Headers

You should be familiar with the most common types of HTTP Headers used in HTTP responses ...
The most important of the response headers is the response code
Response Codes (mandatory)
The first line of the HTTP Response will always contain a response code.
Example response codes indicating success:
HTTP/1.0 200 OK
HTTP/1.0 201 Created

Example response codes indicating failure:
HTTP/1.0 400 Bad Request
HTTP/1.0 500 Server Error

Here is a list of all possible HTTP response codes:
HTTP Status Code Registry


If the service just "GETs" data, the response code to indicate success will usually be 200: OK
If the service just "POSTs" or "PUTs" data, the response code to indicate success will usually be 201: Created.
3XX response codes usually indicate that a networking error of some sort.
4XX response codes usually indicate that an error the occurred because YOU did something wrong.
5XX response codes usually indicate that an error that occurred because something broke on the server.

Client code should be able to handle any response code delivered by the server.

For each service in this specification, the list of possible response codes returned by the service is specified. Client code should be prepared to handle all of these gracefully.
In addition, other response code can be returned by proxy servers, web servers, etc.
Client code should be able to handle these as well.

(2) HTTP Body:

May contain one of a variety of types of content or none at all.
The type of content contained in the body should be specified by the Content-Type header.

The most common Content-Types used in Alpine Restful Services include:
Content-Type: application/json
Content-Type: image/jpeg
Content-Type: image/png

A full listing of all possible Content-Types can be found here:
http://en.wikipedia.org/wiki/MIME_type
