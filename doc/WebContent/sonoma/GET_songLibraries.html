GET songLibraries

Get a list of all song libraries owned by a particular user.


HTTP Request: Headers

GET /songLibraries HTTP/1.1
Host: test.alpineconnectivity.com:8888
Authorization: Basic YWxleGFuZGVyLmoucnlhbkBnbWFpbC5jb206YmVhc3Q=

This service may only be used by registered users who are authorized to use it.
The HTTP Basic Authentication scheme is used to pass the encoded username:password of the user.
Each user may only create, read, update and delete from their own music library.


HTTP Request: Headers: Request Method Parameters

Example

GET /songLibraries HTTP/1.1
Host: test.alpineconnectivity.com:8888
Authorization: Basic YWxleGFuZGVyLmoucnlhbkBnbWFpbC5jb206YmVhc3Q=

NOTE: No parameters need to be passed in the request method header.
The userId is the only parameter needed, and it is extracted from the HTTP Authorization header.


HTTP Response: HEAD: Headers

The following response codes are possible from this service.
(Client code must handle each of them.):

  • 200 Success
  • 400 Bad Request
  • 500 Server Error

HTTP Response: Body

The body of the response should always contain JSON.
The type of JSON returned varies by the response code.
3 types of JSON are possible.

(1) Response Code = 200: "Success"

If the service executed successfully
JSON like the following will be returned in the body of the response:

HTTP/1.1 200 Success

[
    {
        "id": "532b3ae50364bb3e4030ec44",
        "deviceId": "D6735C05A22643B395967FF83CB52251",
        "userId": "alexander.j.ryan@gmail.com",
        "created": 1395342053477,
        "updated": 1395342053488
    },
    {
        "id": "532a0548da067acc93b9e5a1",
        "deviceId": "android-1ac0a0ab-bc49-4b8c-be91-84013fb8d361",
        "userId": "alexander.j.ryan@gmail.com",
        "created": 1395342053466,
        "updated": 1395342053467
    }
]

Description of the data ...

id
(guaranteed)		 The "id" that is returned, is the "song library id".
Most operations on this library must specify this id.
Only the user who created this the song library may request operations on it.
Only a single song library is permitted per device.
deviceId
(guaranteed)		 An identifier that uniquely identifies the device that song library is stored on.
userId
(guaranteed)		 The alpine user who owns all of this.
created
(guaranteed)		 The epoch time that the library was created.
updated
(guaranteed)		 The epoch time that the library was last updated in some way.

(2) Response Code = 400: "Bad Request"

If there was a major error in the request that prevents us from even processing it,
the standard JSON describing the error will be returned in the body of the response:

HTTP/1.1 400 Bad Request

{
    "errorCode": 2,
    "errorMessage": "INVALID PARAMS",
    "data": "userId not specified"
}

(3) Response Code = 500: "Server Error"

If there was a major error on the server,
the standard JSON describing the error will be returned in the body of the response:

HTTP/1.1 500 Server Error

{
    "errorCode": 3,
    "errorMessage": "SERVER ERROR",
    "data": "Unable to connect to the database."
}
