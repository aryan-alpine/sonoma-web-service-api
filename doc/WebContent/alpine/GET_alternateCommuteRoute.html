GET alternateCommuteRoute

Get an alternate travel plan.

Resource

http://dev5:8888/alternateCommuteRoute?start=37.379339,-121.961446&end=37.373596,-122.020500


HTTP Request: Headers

The request method is GET so there is no body that needs to be described.
Only the basic headers are sent.
For example:

GET /alternateCommuteRoute?start=37.379339,-121.961446&end=37.373596,-122.020500 HTTP/1.1
Host: dev5:8888

HTTP Request: Headers: Request Method Parameters

Expected Format:
GET /alternateCommuteRoute?start=[latitude],[longitude]&end=[latitude],[longitude]

start
(required)		 latitude & longitude of the starting point of the route
end
(required)		 latitude & longitude of the end point of the route

HTTP Request: Body

The request method is GET.
So the body does not exist.


HTTP Response: Headers

The following response codes are possible from this service.
(Client code must handle each of them.):

200 Success
400 Bad Request
500 Server Error

If successful (Status Code = 200 Success),
the content of the body of the HTTP Response will be as specified in .
Therefore, the following header will always be sent:
Content-Type: image/jpeg
OR
Content-Type: image/png

If unsuccessful (Status Code = 400 or 500),
the content of the body will be JSON.
Therefore, the following header will always be sent:
Content-Type: application/json
