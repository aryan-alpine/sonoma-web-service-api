POST deviceAccount

Create a device account.


HTTP Request: Headers

The request method is POST.
For example:

POST /deviceAccount HTTP/1.1
Host: test.alpineconnectivity.com:8888

HTTP Request: BODY: Parameters

When the HTTP method is POST or PUT, the RESTful convention is to pass data to a service is via the body of the HTTP request.
This service passes data in this fashion in a JSON data structure.
An example of the data structure and a detailed description of its elements follows...

Example

POST /deviceAccount HTTP/1.1
Host: test.alpineconnectivity.com:8888

{
    "deviceId": "68753A44-4D6F-1226-9C60-0050E4C00067",
    "password": "Z5y334yaksnefovuI83TT"
}

Description

Data in the JSON ...

deviceId
(required)		 An ID that is unique to the device that sent the request.
iPhone device IDs look different than Android device IDs.
In theory the device ID is tied to the hardware of the device and thus will never change.
In practice the device ID is highly unlikely to change.
The device ID sent is guaranteed to be unique across all Alpine applications used on that device. Even if all the applications are un-installed and reinstalled, the device ID will not change.
The only thing that might change the device ID is a complete wiping of the operating system.
password
(required)		 A password that is unique to the device.
This password will probably be randomly generated since the whole purpose of the device account is to give user account-like capabilities without requiring them to select a username + password.
androidRegistrationId
(optional)		 optionally, an android registration id may be specified.
This number may be used to send the user messages on their android device via Google Cloud Messaging.



HTTP Response: HEAD: Headers

The following response codes are possible from this service.
(Client code must handle each of them.):

  • 201 Created
  • 400 Bad Request
  • 500 Server Error

The content of the body of the HTTP Response will always be JSON.
Therefore, the following header will always be sent:
Content-Type: application/json


HTTP Response: Body

If data is returned in the body of the response, it should always come in the form of JSON.
The type of JSON returned varies by the response code.
4 types of JSON are possible.

(1) Response Code = 201: "Created"

If the service executed successfully
JSON like the following will be returned in the body of the response:

HTTP/1.1 201 Created

NOTE: If the response code of 201 was returned, the device account was created.
No data is returned in the body of the HTTP response.

(3) Response Code = 400: "Bad Request"

If there was a major error the request that prevents us from even processing it,
the standard JSON describing the error will be returned in the body of the response:

HTTP/1.1 400 Bad Request

{
    "errorCode": 1,
    "errorMessage": "INVALID JSON",
    "data": "No JSON was POSTed!"
}

(4) Response Code = 500: "Server Error"

If there was a major error the request that prevents us from even processing it,
the standard JSON describing the error will be returned in the body of the response:

HTTP/1.1 500 Server Error

{
    "errorCode": 3,
    "errorMessage": "SERVER ERROR",
    "data": "Error inserting record into the database.."
}
