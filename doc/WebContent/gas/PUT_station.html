PUT station

This service can be utilized in one of 4 different ways:

  • PUT station/[stationId]
    updates gas prices for a particular gas station
  • PUT station/[stationId]/userImage
    updates the userImage for a particular gas station
  • PUT station/[stationId]/thumbnailImage
    updates the thumbnailImage for a particular gas station
    NOTE: This is not to be utilized by client apps and may be discontinued without notice in the future. Don't count on it being there.
  • PUT station/[stationId]/fullImage
    updates the fullImage for a particular gas station
    NOTE: This is not to be utilized by client apps and may be discontinued without notice in the future. Don't count on it being there

Post gas prices at a particular gas stations.

Resource

http://dev5:8888/station


HTTP Request: Headers

The request method is PUT.
For example:

PUT /station HTTP/1.1
Host: dev5:8888
Cache-Control: no-cache

HTTP Request: Headers: Request Method Parameters

id
(required)		 This is the "station" id.
A unique identifier for each gas station.
This will always be an integer and is provided by myGasFeed.com
uuid
(required)		 A universally unique identifier is returned in the HTTP response to this request.
This might be the deviceId.
name
(required)		 The name of the gas station. e.g. Shell, Chevron, 76, etc.
createTime
(required)		 An integer. Milliseconds since the epoch.
reportedBy
(required)		 email address of the person who submitted the data. Needed so that the correct person can be rewarded.
notes
(optional)
lat
(required)		 latitude
lng
(required)		 longitude
address
(optional)
city
(optional)
region
(optional)		 state or province
country
(optional)
telephone
(optional)
prices
(required)		 An array of prices. There must be at least one gas price in this submission.
prices.type
(required)		 The type of gas
prices.price
(required)		 The price of the gas
pics.title
(optional)		 An array of pictures of the gas station. Pictures are entriely optional.
pics.title
(optional)		 The title given to an individual picture.
pics.base64
(optional)		 Pictures are transmitted to the server via the HTTP POST in ASCII base64 encoding.
